# 🛠️ 模型调优与测试 (Model Tuning & Evaluation)

本目录记录了法律查询助手在核心检索与生成环节的测试数据、参数调优过程及多模型表现横向对比，旨在寻找法律场景下“准确率-性能-成本”的最佳平衡点。

## 📑 目录结构

### 1. 🎯 测试集构建 (Test Set Collection)
记录评测基准的构建过程及数据集：
- **《民法典》专业 Benchmark**：基于实务痛点构建的 20 题核心测试集。
- **题目梯度分布**：覆盖显性事实 (60%)、隐性事实映射 (25%)、可解释推理 (10%) 与边界对抗/纠错样本 (5%)。
- **Ground Truth 标定**：精确到具体法条序号的标准召回映射表，用于计算 Recall 与 Precision。

### 2. ⚙️ 策略与参数调优 (Strategy Tuning)
记录不同 RAG 策略与大模型超参数对最终效果的影响：
- **分块策略 (Chunking)**：对比不同 Chunk Size 与 Overlap 长度对法条上下文完整性的影响。
- **检索参数 (Retrieval)**：测试 Top-K 数量对召回率的影响，以及语义向量 (Dense) 与 BM25 (Sparse) 混合检索的最佳权重分配。
- **生成控制 (Generation)**：对比不同参数条件下，模型输出“法言法语”的严谨度及幻觉发生率。

### 3. 🤖 多模型回答对比 (LLMs Comparative Analysis)
横向评测不同基座模型在同一 System Prompt 下的实际表现：
- **参评模型**：DeepSeek V3.1;Doubao-32K;Doubao-1.5-pro-32K
- **评测重点**：
  - **指令遵循度**：是否严格按照“法条引用 -> 法理释明 -> 实务指引”的三段式输出。
  - **安全熔断率**：在模拟“0召回”场景下，模型是否能成功触发拒答机制，不凭空捏造法条。
  - **复杂逻辑表现**：对比各模型在处理“竞合关系”及“优先适用规则”时的法理推演准确度。
