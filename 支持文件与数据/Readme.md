# 📂 基础文件与数据 (Docs & Data)

本目录归档了法律查询助手项目前期的核心产品定义、市场调研与数据基准文件。这些文件构成了整个 Agent 逻辑构建与系统提示词设计的业务支撑。

## 📑 目录结构

### 1. 💡 产品文档 (Product Documentation)
包含项目的核心定义与逻辑框架：
- **PRD (产品需求文档)**：明确了产品“精准法条检索+结构化法理解析”的专业工具定位。
- **Workflow 设计**：涵盖从用户 Query 输入、意图拆解到 RAG 检索及防零召回熔断的完整闭环。
- **Prompt 架构**：针对法律专业场景严格限制边界的三段式 System Prompt 设计。

### 2. 📊 竞品分析 (Competitor Analysis)
包含市面主流垂类产品的基准测试与差异化策略：
- **竞品对标**：对通义法睿、法智等头部产品的深度对比与能力边界摸底。
- **评测基准 (Benchmark)**：基于《民法典》定制的 20 题专业检索测试集，科学配比显性事实(60%)、隐性事实(25%)、逻辑推理(10%)与对抗样本(5%)。
- **功能 Gap 分析**：确立本项目在“语义穿透”与“客观中立”上的核心竞争力。

### 3. 👥 用户调研 (User Research)
包含目标群体的痛点挖掘与需求转化：
- **核心问卷设计**：针对高频法律争议场景与用户行为习惯的精准发问。
- **需求提纯**：将用户在“法律文本晦涩”、“检索效率低”等维度的痛点，转化为产品“只提供法条与释义，不乱做预测”的核心执行标准。Description
